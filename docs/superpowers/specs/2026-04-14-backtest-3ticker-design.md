# Backtest 3종목 비교 기능 설계

**Date:** 2026-04-14  
**File:** `backtest.html`  
**Status:** Approved

## 개요

현재 백테스트 비교기는 두 종목(A, B)만 비교할 수 있다. 세 번째 종목 C를 동적으로 추가하여 최대 3종목을 비교할 수 있도록 기능을 확장한다. UI 전반도 파란/인디고 계열의 깔끔한 디자인으로 개선한다.

---

## 1. UX 흐름 — 동적 추가 방식

- 기본 상태: 종목 A, B 두 칸만 표시
- 종목 B 아래(또는 그리드 하단)에 **"＋ 종목 C 추가하기"** 파란 점선 버튼 표시
- 버튼 클릭 → 초록 계열 종목 C 입력 블록이 슬라이드인 애니메이션으로 나타남
- C 블록 우상단 **✕ 버튼**으로 제거 가능 → 제거 시 다시 + 버튼 표시
- 비교 시작 버튼 클릭 시 C가 비어있으면 2종목, 입력돼 있으면 3종목으로 동작

---

## 2. 색상 체계

| 종목 | 주색 | 배경 | 테두리 | 차트 라인 |
|------|------|------|--------|----------|
| A | `#2563eb` | `#eff6ff` | `#bfdbfe` | `#3b82f6` |
| B | `#ea580c` | `#fff7ed` | `#fed7aa` | `#f97316` |
| C | `#059669` | `#ecfdf5` | `#a7f3d0` | `#10b981` |

---

## 3. UI 개선 — 전체 페이지

현재 회색빛 배경에서 파란/인디고 accent 중심의 깔끔한 디자인으로 개편.

- **배경:** `#f0f4ff` (연한 파란 틴트)
- **카드:** 흰색 배경, 그림자 subtle, border-radius 더 크게 (`20px`)
- **기간 버튼 active:** 인디고 계열 (`#4f46e5`)
- **실행 버튼:** 인디고 그라디언트 (`#4f46e5 → #6366f1`)
- **헤더 배지:** 파란/인디고 그라디언트 유지
- **카드 제목(card-title):** 더 뚜렷한 contrast

---

## 4. 입력 섹션 변경

### 종목 입력 그리드
```
기본(C 없음):  [  종목 A  ] [  종목 B  ]
              [  ＋ 종목 C 추가하기  ]   ← 점선 버튼, 전체 너비

C 추가 후:    [  종목 A  ] [  종목 B  ] [  종목 C ✕  ]
```

- `+` 버튼: 배경 `#f0f4ff`, 테두리 `1.5px dashed #a5b4fc`, 색상 `#6366f1`
  - hover: 배경 `#e0e7ff`, 테두리 `#6366f1`
- C 블록: 초록 계열 (`#ecfdf5` 배경, `#a7f3d0` 테두리, label `#059669`)
- ✕ 버튼: 우상단, `color: #94a3b8`, hover `#dc2626`

### 모바일 (≤600px)
- A, B, C 모두 1열로 수직 쌓기 (`grid-template-columns: 1fr`)
- `+` 버튼 전체 너비

## 5. 결과 — 요약 카드

```
[  카드 A  ] [  카드 B  ] [  카드 C  ]   ← 3열 그리드
```

- 카드 상단 accent bar 색상: A=파랑, B=주황, C=초록
- 카드 내부 구성 (기존과 동일):
  - 종목명 (크게)
  - 🏆 수익 우위 배지 (1등 카드에만)
  - 누적 수익률 (큰 숫자)
  - 투자 원금 / 최종 평가금 / CAGR
- 모바일: 1열 스택

---

## 6. 결과 — 차트

- C 추가 시 Chart.js 데이터셋에 초록 라인 자동 추가 (`#10b981`)
- 범례: A심볼 / B심볼 / C심볼 (동적)
- `switchChartMode()` 함수도 3종목 처리 지원

---

## 7. 결과 — 비교 표

현재: `항목 | A | B | 차이(A−B)`  
변경: `항목 | A | B | C | 차이(A−B)`

- C 컬럼은 C가 추가됐을 때만 표시 (없으면 기존 4열 유지)
- diff 컬럼(A−B)은 기존 그대로 유지
- 추가 비교 행:
  - 누적 수익률 / 투자 원금 / 최종 평가금 / CAGR / 투자기간
- 모바일: 가로 스크롤 (기존과 동일)

---

## 8. 데이터 흐름 변경

```
runBacktest()
  → symA, symB, symC(있으면) 병렬 fetch
  → applyResults(dataA, dataB, dataC?)
    → 요약 카드 2~3개 렌더링
    → 차트 데이터셋 2~3개
    → 비교 표 C 컬럼 유지/숨김
```

- `fetchStockData()` 함수는 변경 없음 (sym, startDate, endDate 인자)
- 기존 변수명 `symA`, `symB` 유지, `symC` 추가
- C 없을 때 `symC = null`로 처리, 결과 렌더 함수에서 null 체크

## 9. 구현 범위 (Out of Scope)

- i18n(다국어) 미적용 (backtest.html은 한국어 전용)
